### HIGH Severity

#### **[H-1] Command Injection via Unquoted PR_NUMBER Variable**

**Location**: `/workspace/devflow/src/claude/commands/devflow/resolve-comments.md:54-56, 74`

**Vulnerability**: The `$PR_NUMBER` variable is used unquoted in multiple command executions after basic sanitization that only removes non-numeric characters. However, the sanitization happens conditionally and the variable can still be used unsafely.

**Code (Lines 54-56)**:
```bash
PR_TITLE=$(gh pr view $PR_NUMBER --json title --jq '.title' 2>/dev/null || echo "Unknown")
PR_URL=$(gh pr view $PR_NUMBER --json url --jq '.url' 2>/dev/null || echo "")
PR_STATE=$(gh pr view $PR_NUMBER --json state --jq '.state' 2>/dev/null || echo "UNKNOWN")
```

**Code (Line 74)**:
```bash
gh pr view $PR_NUMBER --json comments,reviews --json body > /tmp/pr_comments_$PR_NUMBER.json
```

**Attack Scenario**: 
1. If sanitization fails or is bypassed (line 31 only strips non-numerics from `$ARGUMENTS`, but line 41 gets value from `gh pr list` output)
2. Line 41: `PR_NUMBER=$(gh pr list --head "$CURRENT_BRANCH" --json number --jq '.[0].number' 2>/dev/null || echo "")`
3. If `jq` fails or returns unexpected data, PR_NUMBER could contain malicious content
4. Unquoted usage allows command injection: `gh pr view 123; rm -rf / --json`

**Fix**:
```bash
# Always quote variables in command execution
PR_TITLE=$(gh pr view "$PR_NUMBER" --json title --jq '.title' 2>/dev/null || echo "Unknown")
PR_URL=$(gh pr view "$PR_NUMBER" --json url --jq '.url' 2>/dev/null || echo "")
PR_STATE=$(gh pr view "$PR_NUMBER" --json state --jq '.state' 2>/dev/null || echo "UNKNOWN")

# Quote in filename construction
gh pr view "$PR_NUMBER" --json comments,reviews --json body > "/tmp/pr_comments_${PR_NUMBER}.json"
```

**Standard**: CWE-78: OS Command Injection, OWASP A03:2021 - Injection

#### **[H-2] Command Injection via Unquoted REPLY_MESSAGE Variable**

**Location**: `/workspace/devflow/src/claude/commands/devflow/resolve-comments.md:335`

**Vulnerability**: The `$REPLY_MESSAGE` variable contains user-supplied content (comment replies) and is passed unquoted to `gh pr comment` command.

**Code**:
```bash
gh pr comment $PR_NUMBER --body "${REPLY_MESSAGE}"
```

**Attack Scenario**:
1. User crafts reply message containing shell metacharacters
2. Example: `REPLY_MESSAGE="Fixed!'; rm -rf /tmp/*; echo 'Done"`
3. Unquoted `$PR_NUMBER` combined with quoted but potentially dangerous `${REPLY_MESSAGE}` creates injection risk
4. If REPLY_MESSAGE contains backticks, command substitution occurs

**Fix**:
```bash
# Quote PR_NUMBER and use proper quoting for message
gh pr comment "$PR_NUMBER" --body "${REPLY_MESSAGE}"

# OR better: validate reply message doesn't contain dangerous characters
# and use here-string or file-based approach:
gh pr comment "$PR_NUMBER" --body-file <(echo "${REPLY_MESSAGE}")
```

**Standard**: CWE-78: OS Command Injection, OWASP A03:2021 - Injection

#### **[H-3] Command Injection in PR Creation with Generated Content**

**Location**: `/workspace/devflow/src/claude/commands/devflow/pull-request.md:161-175`

**Vulnerability**: PR description generated by sub-agent is embedded in heredoc and passed to `gh pr create`. If sub-agent output contains malicious content (or is compromised), command injection is possible.

**Code**:
```bash
PR_BODY=$(cat <<'EOF'
{generated PR description from sub-agent}
EOF
)

gh pr create \
  --base "$BASE_BRANCH" \
  --head "$CURRENT_BRANCH" \
  --title "{generated PR title}" \
  --body "$PR_BODY" \
  $DRAFT_FLAG
```

**Attack Scenario**:
1. Sub-agent generates PR description containing shell metacharacters
2. Example title: `feat: add auth`; rm -rf /; echo `
3. If title is not properly escaped, command injection occurs
4. Even with quoted variables, backticks in content can execute commands

**Fix**:
```bash
# Use file-based approach to avoid shell interpretation
echo "${PR_BODY}" > /tmp/pr_body_$$.txt
echo "${PR_TITLE}" > /tmp/pr_title_$$.txt

gh pr create \
  --base "$BASE_BRANCH" \
  --head "$CURRENT_BRANCH" \
  --title-file /tmp/pr_title_$$.txt \
  --body-file /tmp/pr_body_$$.txt \
  $DRAFT_FLAG

# Clean up
rm -f /tmp/pr_body_$$.txt /tmp/pr_title_$$.txt
```

**Standard**: CWE-78: OS Command Injection, OWASP A03:2021 - Injection

### MEDIUM Severity

#### **[M-1] Path Traversal in Audit Report File Creation**

**Location**: `/workspace/devflow/src/claude/commands/devflow/code-review.md:60-67`

**Vulnerability**: The `AUDIT_BASE_DIR` is constructed using `${CURRENT_BRANCH}` without sanitization. Malicious branch names could lead to path traversal.

**Code**:
```bash
TIMESTAMP=$(date +%Y-%m-%d_%H%M)
AUDIT_BASE_DIR=".docs/audits/${CURRENT_BRANCH}"
mkdir -p "$AUDIT_BASE_DIR"
```

**Attack Scenario**:
1. Attacker creates branch named: `../../etc/malicious`
2. `AUDIT_BASE_DIR` becomes `.docs/audits/../../etc/malicious`
3. Reports are written outside intended directory
4. Could overwrite system files or expose sensitive paths

**Fix**:
```bash
# Sanitize branch name to prevent path traversal
SAFE_BRANCH=$(echo "$CURRENT_BRANCH" | sed 's/[^a-zA-Z0-9_-]/_/g')
AUDIT_BASE_DIR=".docs/audits/${SAFE_BRANCH}"
mkdir -p "$AUDIT_BASE_DIR"

# Validate the resulting path is within expected directory
REAL_PATH=$(realpath "$AUDIT_BASE_DIR" 2>/dev/null)
EXPECTED_BASE=$(realpath ".docs/audits" 2>/dev/null)
if [[ ! "$REAL_PATH" =~ ^"$EXPECTED_BASE" ]]; then
  echo "❌ Invalid audit directory path"
  exit 1
fi
```

**Standard**: CWE-22: Path Traversal, OWASP A01:2021 - Broken Access Control

#### **[M-2] Insecure Temporary File Usage**

**Location**: `/workspace/devflow/src/claude/commands/devflow/resolve-comments.md:74`

**Vulnerability**: Temporary file created with predictable name in shared directory without proper permissions.

**Code**:
```bash
gh pr view $PR_NUMBER --json comments,reviews --json body > /tmp/pr_comments_$PR_NUMBER.json
```

**Attack Scenario**:
1. Attacker creates `/tmp/pr_comments_123.json` with malicious content before victim
2. Race condition: attacker replaces file between write and read
3. Symlink attack: attacker creates symlink to sensitive file
4. Information disclosure: other users can read PR comments from /tmp

**Fix**:
```bash
# Use mktemp for secure temporary file creation
TEMP_FILE=$(mktemp /tmp/pr_comments_XXXXXX.json) || exit 1
trap "rm -f '$TEMP_FILE'" EXIT

gh pr view "$PR_NUMBER" --json comments,reviews --json body > "$TEMP_FILE"

# Process the temp file
REVIEW_COUNT=$(jq -r '.reviews | length' < "$TEMP_FILE")
COMMENT_COUNT=$(jq -r '.comments | length' < "$TEMP_FILE")

# File automatically cleaned up on exit via trap
```

**Standard**: CWE-377: Insecure Temporary File, CWE-379: Creation of Temporary File in Directory with Insecure Permissions

#### **[M-3] Information Disclosure via Verbose Error Output**

**Location**: `/workspace/devflow/src/claude/commands/devflow/resolve-comments.md:34, 44`

**Vulnerability**: Error messages expose internal variable values that could leak sensitive information.

**Code**:
```bash
echo "❌ Invalid PR number: $ARGUMENTS"
echo "❌ No PR found for branch: $CURRENT_BRANCH"
```

**Attack Scenario**:
1. User supplies malicious input as arguments
2. Error message echoes back the input: `❌ Invalid PR number: $(malicious_command)`
3. Could leak branch names, project structure, or execution context
4. Aids reconnaissance for further attacks

**Fix**:
```bash
# Generic error messages without echoing input
echo "❌ Invalid PR number format. Please provide a numeric PR number."
echo "❌ No PR found for current branch. Create PR first: /pull-request"

# Log detailed error to secure location if needed
echo "[$(date)] Invalid PR argument: $ARGUMENTS" >> .docs/debug/security.log 2>/dev/null
```

**Standard**: CWE-209: Generation of Error Message Containing Sensitive Information, OWASP A04:2021 - Insecure Design

#### **[L-2] No Rate Limiting on GH API Calls**

**Location**: Multiple files using `gh` commands

**Issue**: Commands make multiple `gh` API calls without rate limiting or backoff. Heavy usage could hit GitHub API limits.

**Recommendation**: For future enhancement, add exponential backoff:
```bash
retry_gh_command() {
  local retries=3
  local delay=1
  while [ $retries -gt 0 ]; do
    if "$@"; then
      return 0
    fi
    retries=$((retries - 1))
    sleep $delay
    delay=$((delay * 2))
  done
  return 1
}

retry_gh_command gh pr view "$PR_NUMBER"
```

**Reason not blocking**: Pre-existing pattern; would require broader refactoring.
