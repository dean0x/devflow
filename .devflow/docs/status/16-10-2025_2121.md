## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Three-Layer Architecture**
   - **Layer 1**: TypeScript CLI (installation, configuration management)
   - **Layer 2**: Markdown slash commands (user-facing orchestration)
   - **Layer 3**: Markdown sub-agents (specialized expertise with restricted tools)
   - Why: Separation of concerns, easy to extend, self-documenting
   - Impact: New features = new markdown files, no TypeScript changes needed

2. **Sub-Agents Cannot Invoke Other Sub-Agents**
   - Why: Token optimization, prevent recursive context explosion
   - Impact: Commands orchestrate multiple sub-agents in parallel, not agents themselves
   - Pattern: Orchestrator commands (pre-commit, pre-pr) manage agent execution

3. **Language-Agnostic Design**
   - Pattern: Auto-detect from manifest files, fall back to universal patterns
   - Why: DevFlow should work with any programming language/ecosystem
   - Impact: Agents scan for package.json, Cargo.toml, go.mod, etc. and adapt

4. **Wait-Based Concurrency Control**
   - Pattern: `wait_for_lock_release()` + explicit `wait` commands
   - Why: Prevent race conditions without deleting lock files
   - Impact: All git operations safe, no zombie process issues

5. **Surgical Execution Pattern for Tests**
   - Pattern: Static analysis → Smart selection → Individual execution with limits
   - Why: Large test suites crash Claude Code sessions
   - Impact: Test audits are safe and won't crash sessions
