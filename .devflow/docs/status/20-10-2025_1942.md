## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Command/Agent Separation Pattern**
   - **Pattern**: Simple commands (29 lines) launch specialized sub-agents (347+ lines) that do heavy lifting
   - **Why**: Token efficiency - command loaded every invocation, agent loaded only when needed
   - **Examples**: `/catch-up`, `/commit`, `/research` all follow this pattern
   - **Exception**: `/release` uses orchestrator pattern for user interaction checkpoints
   - **Impact**: ~80% token reduction per command invocation

2. **Orchestrator Pattern for User-Interactive Workflows**
   - **Pattern**: Command handles user interaction flow, agent handles technical implementation
   - **Why**: Agents can't pause mid-execution for user input, commands can orchestrate multi-step confirmations
   - **Example**: `/release` asks for release type, shows preview, requires confirmation before publish
   - **Impact**: Safe, professional release workflow with human checkpoints

3. **Audit Report Standardization**
   - **Pattern**: All audit agents receive `CURRENT_BRANCH`, `AUDIT_BASE_DIR`, `TIMESTAMP` from orchestrator
   - **Why**: Consistent storage, historical tracking, easy comparison across commits
   - **Structure**: `.docs/audits/<branch>/<type>-report.<timestamp>.md`
   - **Fallback**: `.docs/audits/standalone/` for direct agent invocations
   - **Impact**: Organized audit artifacts, branch-specific tracking, timestamp-based history

4. **Template Processing for Configuration**
   - **Pattern**: Read template file, replace placeholders with runtime values, write result
   - **Why**: Enables dynamic configuration without runtime expansion dependencies
   - **Example**: Settings.json tilde (~) replacement with absolute path during init
   - **Impact**: Works around Claude Code's lack of tilde expansion

### Technology Stack

- **Build Tool**: tsc (TypeScript compiler)

### Design Patterns in Use
- **Sub-Agent Pattern**: Commands delegate to specialized agents for focused tasks

- **Template Processing**: Configuration files with dynamic placeholder replacement
- **Convention over Configuration**: Sensible defaults, minimal user setup required

## 💡 Context for Future Developer

### Things to Know

1. **Gotcha #1**: Settings.json tilde expansion
   - Claude Code doesn't expand `~` in settings.json paths
   - Always use absolute paths in settings template
   - Init script does dynamic replacement at install time

2. **Gotcha #2**: Audit agent orchestration
   - Agents receive variables from `/code-review` command via prompt
   - Variables: `CURRENT_BRANCH`, `AUDIT_BASE_DIR`, `TIMESTAMP`
   - Agents must handle both orchestrated (from command) and standalone (direct) invocations

3. **Gotcha #3**: Release agent requires human checkpoints
   - Release workflow pauses before commit, before publish, before tag
   - Command orchestrates confirmations, agent handles technical steps
   - Don't try to make release fully autonomous - safety requires human oversight

4. **Convention #1**: Branch naming
   - Feature branches: `feature/<description>`
   - Fix branches: `fix/<description>`
   - All PRs squash-merged to main with descriptive messages
