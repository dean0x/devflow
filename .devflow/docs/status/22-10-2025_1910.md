## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Command→Agent→Skill Three-Tier Hierarchy**
   - **Commands**: User-invoked orchestrators with full tool access (Task, Write, Read, etc.)
   - **Agents**: Specialized executors running in separate context (Bash, Read, Grep, etc.)
   - **Skills**: Auto-activating quality gates with read-only tools (Task to dispatch agents)
   - **Benefit**: Clear separation of concerns, context efficiency, user control + automation

2. **Dual-Mode Pattern for Research and Debug**
   - **Command Mode**: `/research` and `/debug` for explicit user invocation
   - **Skill Mode**: Research and debug skills auto-activate based on conversation context
   - **Benefit**: Manual control when needed, automatic assistance when detected

3. **Orchestrator Pattern for Commands**
   - **/devlog**: Delegates codebase analysis to project-state agent, focuses on session synthesis
   - **/run**: Orchestrates todo triage → research → implementation workflow
   - **/code-review**: Launches 8 specialized audit agents in parallel
   - **Benefit**: Main session stays clean, heavy work isolated in agent context

4. **Namespace Pattern for CLI Installation**
   - **Structure**: `~/.claude/{commands,agents,skills}/devflow/`
   - **Benefit**: Prevents conflicts with other toolkits, enables multi-toolkit future

### Design Patterns in Use

**Orchestrator Pattern**:
- Commands delegate heavy work to agents
- Main session handles user interaction and synthesis
- Example: /devlog (session context) + project-state agent (codebase analysis)

**Dispatcher Pattern**:
- Skills detect when heavy work needed and auto-launch agents
- Example: Debug skill (119 lines) dispatches to debug agent (475 lines)

**Dual-Mode Pattern**:
- Commands for explicit control, skills for automatic activation
- Same underlying agent invoked by both paths
- Example: /research command + research skill → research agent

**Namespace Pattern**:
- CLI installs to namespaced directories
- Prevents conflicts, enables extensibility
- Example: `~/.claude/skills/devflow/` vs potential `~/.claude/skills/other-toolkit/`
