## 🏗️ Architecture & Design

### Key Architectural Decisions

**1. Command-Agent Dual Pattern**
- **Pattern**: Every command has a corresponding sub-agent
- **Command Role**: Lightweight orchestration (2-3 KB markdown)
- **Agent Role**: Heavy context work (7-12 KB detailed workflows)
- **Benefit**: Keeps main session context clean, offloads analysis to specialized agents

**2. Minimal Orchestration Philosophy**
- **Principle**: Commands hold minimal instructions needed to orchestrate sub-agents
- **Implementation**: brainstorm.md (2.0 KB) delegates to brainstorm agent (7.1 KB)
- **Benefit**: Main Claude Code instance stays focused, agents handle complexity

**3. Separation of Concerns in Planning**
- **Old**: Single `/research` command (monolithic)
- **New**: `/brainstorm` (WHAT approach?) + `/design` (HOW to implement?)
- **Benefit**: Users can choose appropriate granularity for their needs

**4. Skills for Auto-Activation, Commands for Manual Control**
- **Pattern**: `research` skill auto-triggers, no command equivalent anymore
- **Exception**: `debug` has both skill (auto) and command (manual)
- **Decision**: Most skills are auto-only, only debug gets dual-mode

### Technology Stack
