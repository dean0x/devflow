## Architecture & Design

### Key Architectural Decisions (This Session)

**Shepherd Hook Pattern**
A meta-agent pattern using `SubagentStop` hooks to:
- Validate subagent outputs before accepting
- Enforce quality gates (Coder → must pass review)
- Prevent shallow reviews (require file:line references)
- Suggest follow-up actions (chain triggers)

## Final Thoughts

This session marked the completion of the v0.9.0 agent orchestration work and opened a new chapter with hooks research. The shepherd pattern is compelling because it addresses a real gap: DevFlow spawns many subagents but has no automatic validation of their outputs. Hooks provide deterministic enforcement that skills (AI-judged) and commands (user-invoked) cannot.

The key insight is that `SubagentStop` with `type: "prompt"` can make intelligent decisions about whether to accept subagent work, potentially blocking until quality gates pass. This could transform DevFlow from "recommended best practices" to "guaranteed enforcement."

**Warning**: Don't over-engineer the hooks. Start with one pattern (Coder → Review gate) and validate it works before expanding. The hybrid approach (fast bash pre-filter, escalate to prompt) might be premature optimization.
