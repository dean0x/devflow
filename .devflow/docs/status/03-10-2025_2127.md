## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **CLI as Package Installer**
   - Why: Separates installation mechanism from runtime commands
   - Impact: Users install once, commands persist in Claude Code configuration
   - Location: `src/cli/` contains TypeScript installer, `src/claude/` contains runtime assets

2. **Markdown-Based Commands and Agents**
   - Why: Self-documenting, easy to modify, human-readable
   - Impact: No compilation needed for command changes, AI agents can easily understand and modify
   - Location: `src/claude/commands/devflow/*.md` and `src/claude/agents/devflow/*.md`

3. **Modular Sub-Agent Architecture**
   - Why: Token optimization through specialized contexts, parallel execution capability
   - Impact: Sub-agents cannot invoke other sub-agents (prevents recursion), focused expertise
   - Design constraint: Sub-agents are isolated by design (documented in CLAUDE.md)

4. **Git-First Documentation Structure**
   - Why: Sessions are ephemeral, git provides persistence
   - Impact: `.docs/` directory for session tracking, all important context goes in git
   - Location: `.docs/status/` for session logs, `.docs/audits/` for audit reports

- **Runtime**: Claude Code environment (user's ~/.claude/ directory)
