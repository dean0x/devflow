### Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Single plugin, not marketplace | DevFlow is one cohesive toolkit with interdependent commands | Marketplace with modular plugins (rejected: adds complexity, fragments UX) |
| Keep CLI working | CLI provides "batteries included" experience (CLAUDE.md, .claudeignore, .docs/) | Plugin-only (rejected: loses CLAUDE.md installation capability) |
| Prefix skills with `devflow-` | Enables reliable cleanup on uninstall without hardcoding list | No prefix (rejected: can't distinguish from other plugins' skills) |
| User-facing docs show clean names | Prefix is implementation detail users don't need to know | Show prefixed names (rejected: clutters documentation) |
