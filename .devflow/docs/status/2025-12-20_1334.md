## Work Completed

### 1. Renamed SwarmOrchestrator to Coordinator
- Renamed `swarm-orchestrator.md` → `coordinator.md` for both agent and command
- Updated all internal references and state paths

### 2. Created Planning Orchestration Layer
Mirroring the Swarm/Coordinator execution pattern, created a planning layer:

**Specifier** (single feature specification):
- Transforms rough feature idea into well-defined GitHub issue
- Uses native Explore + Plan subagents for codebase understanding
- Batches related questions for efficient user interaction
- Auto-creates GitHub issues after clarification (no confirmation step)
- Flow: `UNDERSTAND → EXPLORE → PLAN → CLARIFY → SPECIFY → CREATE ISSUE`

**Planner** (multi-feature orchestrator):
- Spawns Specifiers in parallel for each feature
- Synthesizes dependency graph across features
- Creates release issue with execution waves
- Flow: `PARSE → SPECIFY (parallel) → SYNTHESIZE → CREATE RELEASE → REPORT`

### 3. Simplified Swarm Review Handling
- Removed auto-fix review loop from Swarm
- Swarm now stops after CodeReview and reports findings
- Comment resolution is explicit via `/resolve-comments`
- Rationale: Separates AI quality gate from human PR review

### 4. Cleaned Up Redundant Commands
Removed commands now covered by orchestration:
- `design.md` - covered by Design agent in Swarm
- `plan.md` - covered by Plan subagent
- `get-issue.md` - covered by Coordinator workflow

Kept for standalone use:
- `breakdown.md` - quick task breakdown without full orchestration
- `run.md` - direct implementation without design phase

---

## Architecture Summary

```
PLANNING LAYER
Planner → Specifier(s) → GitHub Issues
                ↓
EXECUTION LAYER
Coordinator → Swarm(s) → Design → Coder → CodeReview → PRs
                                                         ↓
                                               /resolve-comments
```

### Pattern: Two-Layer Orchestration
| Layer | Single Unit | Multi-Unit Orchestrator |
|-------|-------------|------------------------|
| Planning | Specifier | Planner |
| Execution | Swarm | Coordinator |
