## 🏗️ Architecture & Design

### Key Architectural Decisions
The Git agent uses **operation-based dispatch** pattern:

```
Git Agent
├── fetch-issue    → Fetch GitHub issue details
├── comment-pr     → Create PR inline comments
├── manage-debt    → Update tech debt backlog
└── create-release → Create GitHub release
```

All operations share patterns from `devflow-github-patterns` skill:
- Rate limiting (1-2s delays, 60s wait if <10 remaining)
- Comment deduplication algorithms
- Line-in-diff validation
- Issue data parsing

### Technology Stack
| Component | Details |

| Runtime | Node.js >=18.0.0 |
| Language | TypeScript 5.3.3 |
| CLI Framework | Commander.js |
| Package | devflow-kit v0.9.0 |

### Design Patterns in Use
- **Tiered Skills Architecture** - Foundation → Pattern → Specialized → Domain skills
- **Parameterized Agents** - Single agent with operation dispatch
- **Iron Laws** - Non-negotiable principles per skill
- **Orchestrator Pattern** - Commands spawn agents, don't do work themselves
