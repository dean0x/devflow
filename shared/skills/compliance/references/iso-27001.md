# ISO/IEC 27001 — Code-Level Checks

**Standard**: ISO/IEC 27001:2022 Annex A — Information Security Controls

## Relevant Annex A Controls as Code Checks

### A.8 — Technological Controls (Information Asset Handling)

| Control | Code obligation |
|---------|----------------|
| A.8.2 Information classification | Annotate data models with classification level (public / internal / confidential / restricted) |
| A.8.3 Information handling | Confidential/restricted data must be encrypted at rest and in transit |
| A.8.10 Information deletion | Secure deletion procedures for regulated data; overwrite or cryptographic erasure |
| A.8.24 Cryptography | Approved algorithms only (AES-256-GCM, RSA-4096+, ECDSA-P256+); no MD5/SHA-1 for security |
| A.8.25 Secure development lifecycle | Security review gates in the CI/CD pipeline |

### A.12 — Operations Security (Monitoring and Logging)

| Control | Code obligation |
|---------|----------------|
| A.12.4.1 Event logging | Log privileged access, authentication events, and data access to regulated stores |
| A.12.4.2 Protection of log information | Logs must be append-only or write to a tamper-resistant sink; no log deletion by app code |
| A.12.4.4 Clock synchronization | All log timestamps from NTP-synchronized source; UTC ISO-8601 format |

### A.14 — System Acquisition, Development, and Maintenance

| Control | Code obligation |
|---------|----------------|
| A.14.1.2 Securing application services | TLS enforced; no HTTP fallback for services handling regulated data |
| A.14.2.1 Secure development policy | Documented secure coding standards referenced in contributing guide |
| A.14.2.5 Secure system engineering principles | Defense in depth; fail secure; least privilege |
| A.14.2.8 System security testing | Security tests in CI; dependency scanning; SAST for confidential-data code paths |

## Classification Annotation Pattern

```typescript
/** @classification confidential — contains employee PII */
interface EmployeeRecord {
  id: string;
  name: string;       // PII
  nationalId: string; // PII — restricted
  department: string; // internal
}
```

## Common Code-Level Gaps

- No data classification in model definitions (A.8.2)
- Logs written to the same sink as application output — no tamper protection (A.12.4.2)
- Development environment using production data without de-identification (A.14.2.8)
- Deprecated cryptographic primitives (MD5, SHA-1) used for data integrity (A.8.24)
